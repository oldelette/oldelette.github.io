Basic Concept

Python在例外處理的機制，包含了：

  • 基本的例外錯誤處理(try-except)
  • 不同的例外錯誤處理(different exceptions)
  • finally區塊(try-except-finally)
  • 自行拋出例外錯誤(raise exceptions)

Try-Except

如果在開發的過程中，大概知道可能會發生的例外錯誤類別，則可於except關鍵字之後加上該類別名稱，而發生此類別以外的錯誤，就會執行沒有加類別名稱的except區塊

try:
    # 你要執行的程式
except ValueError:
    # 發生ValueError時要執行的程式
except ZeroDivisionErro:
    # 發生ZeroDivisionErro時要執行的程式
except:
    # Other error

finally、else

ref
完整的格式顺序是：try —> except X —> except —> else—> finally

  • 如果 else 和 finally 都存在的话，else 必須在 finally 之前，finally 必須在整個程序的最後。
  • else 的存在是以 except 的存在為前提












try:
    Nomal execution block
except A:
    Exception A handle
except B:
    Exception B handle
except:
    Other Exception handle
else:
    if no exception, get here
finally: 
    print('finally')

exception as

在Python 3中，可以在except捕捉到例外後，將例外物件指定給變數







try:
    num1 / num2
except ZeroDivisionError as error:
    print(type(error))
except Exception as error:
    print(type(error),str(error))

印出

<class 'ZeroDivisionError'>
<class 'TypeError'> unsupported operand type(s) for /: 'int' and 'NoneType'

basicConfig

logging 提供了 basicConfig 的 method 來做一些基本的設定

logging.basicConfig(
    level=logging.INFO, 
    filename=log_filename, 
    filemode='w',
    format='[%(levelname)1.1s %(asctime)s %(module)s:%(lineno)d] %(message)s',
    datefmt='%Y%m%d %H:%M:%S',
)
  • filename: 可以設定要寫入的檔案路徑，如果沒有設定時，預設會輸出到 sys.stderr，也就是 console。
  • level: 先前提到的 level 預設是 WARNING，我這邊修改成 INFO。
  • format: 這邊可以制定 log 訊息的格式
  • datefmt: 預設的時間格式
  • asctime: 指的是 log 產生的時間，預設格式是 2003-07-08 16:49:45,896。

Formatter

LogRecord attributes

format='%(filename)s:%(asctime)s:%(levelname)s:%(message)s')

列出常用的參數

Attribute name Format Description
asctime %(asctime)s Human-readable time when the LogRecord was created. By default this is of the form ‘2003-07-08 16:49:45,896’
filename %(filename)s Filename portion of pathname.
name %(name)s Name of the logger used to log the call.
levelname %(levelname)s Text logging level for the message (‘DEBUG’, ‘INFO’, ‘WARNING’, ‘ERROR’, ‘CRITICAL’).
message %(message)s The logged message, computed as msg % args. This is set when Formatter.format() is invoked.
  • name 顯示哪個 module

假設 log.py 有 import employee.py， 那執行 python log.py 時，寫在 employee.py 裡面的 Log file name 為 “employee” ， 而在 log.py 裡面的 log file name 為 root。

但是如果直接執行 python employee.py ， 則在 employee.py 裡面的 log file name 是 __main__

Handler

1. Create handlers

console_handler = logging.StreamHandler()
file_handler = logging.FileHandler(‘handler.log’)
time_rotating_handler = logging.handlers.TimedRotatingFileHandler(‘time-rotating-handler.log’)

  • logging.StreamHandler(): 預設是輸出到 sys.stderr。
  • logging.FileHandler(): 可將 log 寫入到檔案中。
  • logging.handlers.TimedRotatingFileHandler(): 也是將 log 寫入到檔案中，但是可以額外做 log rotation 的功能，詳閱。

2. Create formatters

制定 log 訊息的格式

console_format = logging.Formatter(‘%(asctime)s - %(name)s - %(levelname)s - %(message)s’)
file_format = logging.Formatter(‘%(levelname)s - %(message)s’)
time_rotating_format = logging.Formatter(‘%(asctime)s - %(name)s - %(levelname)s - %(message)s’)

3. Set the format for handlers

將制定的log 訊息格式加入 handler 中

console_handler.setFormatter(console_format)
file_handler.setFormatter(file_format)
time_rotating_handler.setFormatter(time_rotating_format)

4. Set the log level for handlers:

設定 handler 的 log level

console_handler.setLevel(logging.INFO)
file_handler.setLevel(logging.INFO)
time_rotating_handler.setLevel(logging.INFO)

5. Get a logger:

取得 logger。

logger = logging.getLogger(__name__)

6. Add the handlers to the logger

把第 1 項到第 4 項所設定的 handler 加入到前一項的 logger 中

logger.addHandler(console_handler)
logger.addHandler(file_handler)
logger.addHandler(time_rotating_handler)

Logging configuration

第一個方式對於單一功能與單一格式的 log 或許就夠用了，第二個方法雖然可以應付多種不同格式的 log，但是 set 來 set 去的讓整個程式碼看起來很複雜。

於是 logging 提供了更多元的 Logging configuration，這邊主要支援兩種設定方式，dictConfig() 以及 fileConfig()

說明一下 SETTING 這個 dict 的設定：

  • formatters: 這裡可以設定多種格式的 log 訊息，裡面 format 的設定方式與第一節介紹相同。
  • handlers: 這裡可以設定多個 handler。
  • class: 這邊是選擇 handler 的型態，同上一節的介紹。
  • formatter: 這邊選擇上一項所設定的格式
  • loggers: 這邊可以設定多個 logger。
  • handlers: 在同一個 logger 裡面可以選擇一個或是多個 handler。
  • level: 設定這一個 logger 的 log lever。

Itertools

Functions creating iterators for efficient looping.

Iterator是常用来做序列的對象，只有當選到某個值的时候，才會進行計算出這個值

Infinite iterators

Iterator Arguments Results
count() start, [step] start, start+step, start+2 * step, …
cycle() p p0, p1, … plast, p0, p1, …
repeat() elem [,n] elem, elem, elem, … endlessly or up to n times

Example

















import random
from itertools import chain
from random import choice, choices

def generate_password(length: int):
    digits = "0123456789"
    letters = "".join(chr(x) for x in chain(
        range(ord("a"), ord("z") + 1),
        range(ord("A"), ord("Z") + 1)
    ))
    total = digits + letters
    password = "".join(chain([choice(digits)], choices(total, k=length - 1)))
    
    return password

print(generate_password(10))

Python Typing

typing 是 Python 3.5 以上內建的模組

Python 內建的型別（例如： int, float, dict, list, …）可以直接使用型別註釋，但如果是較複雜的資料型別就需要 typing 模組的輔助。

typing import

中文文章 ref
typing — Support for type hints

typing.List

表達都是 int 的列表(list)， 如果用下面方法會出現 TypeError: ‘type’ object is not subscriptable

def test(x: list[int]):
    pass

正確的方式是使用 typing 模組的 List 進行表達

from typing import List

def test(x: List[int]):
    pass

typing.Union

Python 的動態型別，使得我們可以讓一個函式或方法能夠接受多種型別。

  • 下面 Function 可同時接受字串與數字







from typing import Union

def print_something(x: Union[int, str]):
    if isinstance(x, (int, str, )):
        print(f'Got {x}')
    else:
        raise TypeError('Only int & str are accepted')

Fliter

Python Functional Programming 實作教學 — filter()篇

在 python 中的 filter 函數就是用來過濾像是陣列 ( list ) 之類的可疊代對象 ( Iterable )，將不符合條件的值一一篩選掉。

filter(<function>, <Iterable>)

Example1

取出狀態 = pending 的 info

如果用直觀寫法(20行)，除了列出 status = ‘pending’ 還要特定日期 dueDate = ‘2019-05-21’ 的 task 等等資訊… 日後若又有新的 filter 想要列出特定 id，這會越來越複雜，而且也無法一目瞭然這幾行 code 再幹嘛。
每當增加一個需求就必須改寫一次 Filter

在 FP 的世界裡，要把部分邏輯交由外部的使用者決定是很簡單的，我們只要要求使用者傳一個 function 進來，並預期這個 function 會回傳某種值就可以了 (22/23行)。

Reduce

reduce()函式即為化簡函式，
它的執行過程為：每一次迭代，都將上一次的迭代結果 （注：第一次為init元素，如果沒有指定init則為seq的第一個元素）與下一個元素一同傳入二元func函式中去執行。
在reduce()函式中，init是可選的，如果指定，則作為第一次迭代的第一個元素使用，如果沒有指定，就取seq中的第一個元素。

from functools import reduce

Example - 階層

Python中並沒有給出一個求階乘的內建函式，拿reduce函式來實作。







from functools import reduce

def mult(x,y):
    return x*y
# 5!
sum1 = reduce(mult,range(1,6))
