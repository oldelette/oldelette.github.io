Inotify

inotify - monitoring file system events

Inotify 是取用 linux 底下的系統自動監聽檔案變化的方式，可以實現基於inode的檔案監控。
也就是說監控物件不再侷限於目錄，也包含了檔案

The inotify API provides a mechanism for monitoring file system events.
Inotify can be used to monitor individual files, or to monitor directories.
When a directory is monitored, inotify will return events for the directory itself, and for files inside the directory.

  • Disadvantage
    inotify 無法監控軟連結型的子目錄。

Inotify Function

The following system calls are used with this API:

  • inotify_init(void):
    建立一個 inotify例項

  • inotify_add_watch(int fd, const char *pathname, uint32_t mask):
    加入檔案或目錄到inotify進行監測

    • fd ：就是通過 inotify_init 函式建立的 inotify 控制代碼。
    • path ：要監聽的檔案或目錄的路徑。
    • mask ：要監聽的事件

    inotify_add_watch()會回傳一個監控器（watch descriptor），它代表pathname指向的inode(不同的pathname有可能指向相同的inode)。

  • inotify_rm_watch(int fd, int wd):
    移除一個watcher

    • fd ：呼叫 inotify_init 函式返回的 inotify 控制代碼。
    • wd ：由 inotify_add_watch 函式的返回被監聽檔案或目錄的描述符。

讀取變動事件

inotify 並沒有提供特定的介面來獲取被監聽的檔案或目錄的變動事件，而是通過通用的 read 函式來讀取

int read(int fd, void *events, size_t len);
  • fd ：由 inotify_init 建立的 inotify 控制代碼。
  • events ：存放變動事件的緩衝區。
  • len ：緩衝區的大小。

Each successful read returns a buffer containing one or more of the following structures:

struct inotify_event {
    int      wd;       /* Watch descriptor */
    uint32_t mask;     /* Mask of events */
    uint32_t cookie;   /* Unique cookie associating related
                          events (for rename(2)) */
    uint32_t len;      /* Size of name field */
    char     name[];   /* Optional null-terminated name */
};

Inotify Events

  • Inotify 可以監控的文件系统包括：
Inotify Events Description
IN_ACCESS 檔案的讀操作
IN_MODIFY 文件被 write
IN_ATTRIB 文件屬性被修改，如chmod、chown、touch等
IN_CLOSE_WRITE 可寫文件被 close
IN_CLOSE_NOWRITE 不可寫文件被close
IN_OPEN 文件被open
IN_MOVED_FROM 文件被移走,如mv
IN_MOVED_TO 文件被移過來，如mv、cp
IN_CREATE 創建新文件
IN_DELETE 文件被删除，如rm
IN_DELETE_SELF 自删除，即一個可執行文件在執行時刪除自己
IN_MOVE_SELF 自移動，即一個可執行文件在執行时移動自己
IN_UNMOUNT 宿主文件系統被 umount
IN_CLOSE 文件被關閉，等同於 (IN_CLOSE_WRITE | IN_CLOSE_NOWRITE)
IN_MOVE 文件被移動，等同於 (IN_MOVED_FROM | IN_MOVED_TO)

pyinotify

Monitor Filesystem Events with Pyinotify

pyinotify 就是 Inotify 的 python 版本實作

Similar to events in inotify, the Python implementation will be through an EventProcessor object with method names containing “process_” that is appended before the event name.
For example, IN_CREATE in pyinotify though the EventProcessor will be process_IN_CREATE.
The table below lists the inotify events used in this guide.
In depth descriptions can be found in the man pages of inotify.

EventProcess

Python implementation will be through an EventProcessor object with method names containing “process_” that is appended before the event name

Python 實作 Class EventProcess 的方法 為 process_ 開頭加上 Inotify Events.
例如:
IN_CREATE in pyinotify though the EventProcessor will be process_IN_CREATE

class MyEventHandler(pyinotify.ProcessEvent):
        
    def process_IN_ACCESS(self, event):
        print("ACCESS event:", event.pathname)

    def process_IN_ATTRIB(self, event):
        print("ATTRIB event:", event.pathname)

    def process_IN_CLOSE_NOWRITE(self, event):
        print("CLOSE_NOWRITE event:", event.pathname)

    def process_IN_CLOSE_WRITE(self, event):
        print("CLOSE_WRITE event:", event.pathname)

    def process_IN_CREATE(self, event):
        print("CREATE event:", event.pathname)

    def process_IN_DELETE(self, event):
        print("DELETE event:", event.pathname)

Create a Watch Manager

Create notify_me.py in a text editor.



import pyinotify
watch_manager = pyinotify.WatchManager()

Create an Event Notifier

Instantiate the Notifier class with an instance of WatchManager as the first argument and a ProcessEvent subclass instance as the second argument.

notifier = pyinotify.Notifier(watch_manager, MyEventHandler())

Add a Watch

A watch is a file or directory to be monitored by pyinotify. Create a sample directory called notification_dir in your home directory:

mkdir ~/notification_dir

Add this directory to our file system notification system. Call add_watch() on the watch manager instance watch_manager.




import os
watch_this = os.path.abspath("notification_dir")
watch_manager.add_watch(watch_this, pyinotify.ALL_EVENTS)

Start the Watch

By looping the Notifier object, the directory can be monitored continuously

event_notifier.loop()

loop(self, callback=None, daemonize=False, **args)

Ref
Events are read only one time every min(read_freq, timeout) seconds at best and only if the size to read is >= threshold.
After this method returns it must not be called again for the same instance.

可選 Parameters:

  • callback (callable object or function) - Functor called after each event processing iteration.
  • daemonize (boolean) - This thread is daemonized if set to True.
  • args (various) - Optional and relevant only if daemonize is True.

Inode

Linux和其他類Unix作業系統通過將所有內容都視為檔案（甚至是硬體裝置）來保持一致性。
鍵盤，滑鼠，印表機，顯示器，硬碟，程式，甚至目錄都被視為 Linux中 的檔案

每個 Linux檔案或目錄都有一個inode，而這個 inode包含了所有檔案的後設資料 (也就是說，讀取檔案所需的管理資料都儲存在inode中)。
例如，inode包含儲存檔案的所有塊的列表，該檔案所有者資訊、許可權以及為該檔案設定的所有其他屬性。

inode table

在檔案管理上，有三個重要的表格，分別是:

  1. 在記憶體中的inode table
  2. 在硬碟中的inode table
  3. 在記憶體中的file table

當使用open() 打開檔案，若是記憶體中的inode table可以找到對應的inode，那就可以直接對資料進行存取，如果找不到的話，就會到硬碟中的inode table找尋相對應的inode並且讀取到記憶體中。

inode structure

  • Inode number
  • Mode information to discern file type and also for the stat C function
  • Number of links to the file
  • UID of the owner
  • Group ID (GID) of the owner
  • Size of the file
  • Actual number of blocks that the file uses
  • Time last modified
  • Time last accessed
  • Time last changed

Command Line

  • 顯示指定檔案資訊
    可以使用 stat命令顯示某個檔案或目錄的inode資訊，必須指定檔案或目錄名

  • 列印檔案的inode number
    ls -i 說明需要顯示每個檔案的 inode number

  • 顯示檔案系統inode的使用資訊
    df -i --inodes 選項來接收有關可用和已使用的 inode
    • Inodes: 可用之 inode
    • IUsed: 已使用之 inode

System call

System Call 的目的是作為執行中user process與kernel溝通的介面，代表OS可以提供的服務項目。
當 process執行時，若需要OS提供某種服務(例如Disk read txt內容)
process會issue Trap，帶入system call ID及其相關參數 (例如invoke the required system call)
當OS收到此請求後，會去執行對應的system call，當完成後，再將結果通知傳回給process。

system call 是 process 和 OS 之間的介面，當使用者程式需要 OS 的服務時，使用者程式便去呼叫 system call

Type

類型 內容 範例
