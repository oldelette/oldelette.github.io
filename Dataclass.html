Complex Initialization

Consider a scenario, where you want an attribute to be a list upon initialization. How do we make it happen?

One simple way would be do that in post_init method.














import random
from typing import List

def get_random_marks():
    return [random.randint(1,10) for _ in range(5)]

@dataclass
class Student:
    def __post_init__(self):
        self.marks = get_random_marks()

a = Student()
print(a.marks)  #[1, 6, 1, 1, 3]

dataclasses.field

python dataclasses.field (*, default=MISSING, default_factory=MISSING, repr=True, hash=None, init=True, compare=True, metadata=None)

如果不想使用 __post_init__ 做變數的初始化，可以使用 dataclasses.field

Python coredevs have a solution for us.
It is possible to customize behaviour of dataclass fields and their impact on the dataclass using dataclasses.field




@dataclass
class Student:
    marks: List = field(default_factory=get_random_marks)

dataclasses.field accepts a default_factory argument which can be used to initialize the field if a value is not passed at the time of creation of the object.

default_factory should be a callable( generally a function ) which accepts no arguments.

repr

如果為 True（默認值），則該字串包含在生成的 __repr__() 方法返回的字符串中。

The auto-generated __repr__ method uses all fields for representation.
Well, this is not an ideal situation in a lot of cases. Specially, when your dataclass has a lot of fields.
A single object representation would get pretty huge, and a debugging nightmare.














@dataclass(order=True)
class User:
	name: str = field(compare=False)
	age: int = field(repr=False) 
    # This tells the dataclass to not show age in the representation
	height:float = field(repr=False)
	weight:float = field(repr=False)
	city:str = field(repr=False, compare=False)
    #Do not use city for representation and comparison
	country:str = field(repr=False, compare=False)

a = User("John Doe", 24, 1.7, 70, "Massachusetts", "United States of America")
b = User("Adam", 24, 1.6, 65, "San Jose", "United States of America")

output is:

User(name='John Doe')
User(name='Adam')

Creating nested dataclass objects in Python

















from dataclasses import dataclass, is_dataclass

def nested_dataclass(*args, **kwargs):
    def wrapper(cls):
        cls = dataclass(cls, **kwargs)
        original_init = cls.__init__
        def __init__(self, *args, **kwargs):
            for name, value in kwargs.items():
                field_type = cls.__annotations__.get(name, None)
                if is_dataclass(field_type) and isinstance(value, dict):
                     new_obj = field_type(**value)
                     kwargs[name] = new_obj
            original_init(self, *args, **kwargs)
        cls.__init__ = __init__
        return cls
    return wrapper(args[0]) if args else wrapper

from dataclasses import dataclass
from typing import ClassVar

@dataclass
class Foo:
    ivar: float = 0.5
    cvar: ClassVar[float] = 0.5
    nvar = 0.5

foo = Foo()
Foo.ivar, Foo.cvar, Foo.nvar = 1, 1, 1
print(Foo().ivar, Foo().cvar, Foo().nvar)   # 0.5 1 1
print(foo.ivar, foo.cvar, foo.nvar)         # 0.5 1 1

Instance variables 物件變數 V.S. Class variables 類別變數

Instance variables 物件變數
簡單地說，instance variables 物件變數就是在物件層級的變數，也就是初始實體化物件的時候所賦予它的變數，當物件還存在時，你可以再賦予它另外的變數，也可以刪除它的變數，這就是所謂的 instance variables 物件變數。

物件變數還有另一個特點，那就是每個物件的 instance variables 是不相關聯的，也就是互相獨立。

Class variables 類別變數
顧名思義，class variables 類別變數是指變數是在 class 類別層級，它是在 class 類別的結構中，要存取 class variables 的話，則要從 class name 類別名稱來著手。

Class variables 類別變數的主要用途是當作 meta information (原生訊息) 或是做為物件間的共同資訊。

Instance Attribute

需伴隨物件(Object)的生成來建立，也就是透過點(.)的語法或在建構式(Constructor)中所生成的屬性(Attribute)。
各物件(Object)的實體屬性(Instance Attribute)各自獨立，修改某一個物件(Object)的實體屬性(Instance Attribute)值時，不會影響到其他物件。

透過點(.)的方式來建立物件(Object)的屬性(Attribute)，mazda和toyota物件的color與seat為 實體屬性(Instance Attribute)，互相獨立且有各自的屬性值。

Property 特性二：

特性二：重新實現一個屬性的 setter、getter 和 deleter 方法

以下是 @property 的 setter、getter 和 deleter 的使用範例：

















class Bank_acount:
    def __init__(self):
        self._password = '預設密碼 0000'

    @property
    def password(self):
        return self._password

    @password.setter
    def password(self, value):
        self._password = value

    @password.deleter
    def password(self):
        del self._password
        print('del complite')

class Employee:

    def __init__(self,first,last):
        self.first = first
        self.last = last
    @property
    def email(self):
        return '{}.{}@email.com'.format(self.first,self.last)
    @property
    def fullname(self):
        return '{} {}'.format(self.first,self.last)

    @fullname.setter
    def fullname(self,name):
        first, last = name.split(' ')
        self.first = first
        self.last = last

    @fullname.deleter

        print('Delete Name!')
        self.first = None
        self.last = None

Instance Method

Python 類別(Class)中沒有加任何裝飾詞(Decorator)的方法(Method)，至少要有一個self參數，於方法 (Method)被呼叫時指向物件 (Object)

  • 透過 class 呼叫 Instance method 時，會使得 python 編譯器無法將 self 參數指向物件 (Object)，而發生 TypeError。
  • Instance Method 要利用 self.__class__屬性(Attribute) 來改變door類別屬性(Class Attribute)。








class Cars:
    # 建構式
    def __init__(self):
        self.color = "blue"
    # 實體方法(Instance Method)
    def drive(self):
        print(f"{self} is {self.color}.")
Cars.drive()

TypeError: drive() missing 1 required positional argument: 'self'

Class Method

Python類別(Class)中有 @classmethod裝飾詞(Decorator)的方法(Method)，被呼叫時，相較於實體方法(Instance Method)的self參數指向物件(Object)，類別方法(Class Method)為cls參數，指向類別(Class)

類別方法 Class Method 又稱為工廠方法 Factory Method

  • 當 Mehod 被呼叫時， Python 編譯器會傳入 Class，使得 cls 參數指向該類別，接著透過 cls 參數來存取類別屬性 Attribute。










# 汽車類別
class Cars:
    door = 4  # 類別屬性
    # 類別方法(Class Method)
    @classmethod
    def open_door(cls):
        print(f"{cls} has {cls.door} doors.")
mazda = Cars()
mazda.open_door()  #透過物件呼叫
Cars.open_door()  #透過類別呼叫

Static Method

Python類別中有 @staticmethod裝飾詞(Decorator)的方法(Method)，可以接受任意的參數，也因為它沒有self及cls參數，所以靜態方法(Static Method)無法改變類別(Class)及物件(Object)的狀態。

  • 來源端不論透過 類別(Class)或 物件(Object)皆可呼叫，Python編譯器於執行期間(Runtime)不會傳入 self 及 cls 參數至靜態方法(Static Method)。













# 汽車類別
class Cars:
    # 速率靜態方法
    @staticmethod
    def speed_rate(distance, minute):
        return distance / minute
# 透過物件呼叫
van = Cars()
van_rate = van.speed_rate(10000, 20)
print("van rate: ", van_rate)
# 透過類別呼叫
sports_car_rate = Cars.speed_rate(20000, 20)
print("sports car rate: ", sports_car_rate)

透過一般的方式，裝飾器可能會以下方的形式使用：

def p_decorate(func):
   def func_wrapper(name):
       return "<p>{0}</p>".format(func(name))
   return func_wrapper

def strong_decorate(func):
    def func_wrapper(name):
        return "<strong>{0}</strong>".format(func(name))
    return func_wrapper

def div_decorate(func):
    def func_wrapper(name):
        return "<div>{0}</div>".format(func(name))
    return func_wrapper

淺談Python閉包和late binding機制

a closure (also lexical closure or function closure) is a technique for implementing lexically scoped name binding in a language with first-class functions

Closure 有很廣泛的應用，我們可以用閉包來減少全局變量的定義進而提供數據隱藏，還可以提供面向對象的編程風格，使代碼實現更優美。

先了解 嵌套函數 (nested function) 和非局部變量 (non-local variable) 的概念

  • nested function: 定義在函數內部的函數
  • non-local variable: 在 nested fuction 訪問外層函式的 local variable 稱為 non-local variable.

printer 是一個 nested function， msg 就是 printer 的 non-local variable。










def print_function(msg):
	# printer is a nested function
    def printer():
        print(msg)
    return printer
    
p = print_function("Hello world!")
p()
print("p: ",p.__closure__)

return printer 語句返回的不僅僅是一個函數，還有一個閉包，這個閉包裏保存了函數要用到的所有non-local variable (非局部變量)

Binding

late binding 機制

Python’s closures are late binding. This means that the values of variables used in closures are looked up at the time the inner function is called.

LEGB

根據 Python.org 的 tutorial 中的說明，當一個變數被使用時，會遵循 LEGB 的規則，也就是 Local、Enclosing、Global 與 Builtins







glob = 3
def func(x):
    y = x + glob
    def inner():
        return y + 1
    return inner, abs
  • Local，在 local variables 裡查找。
  • Global，是 global variable。
  • Enclosing，也是 enclosing scope。
  • Builtins，也就是從 builtins 模組裡去找，上面的例子裡就是最後用到的 abs 。

當以上都找不到這個你要的變數時，就會引發 NameError 。
