Web basic

Web 運作流程

step1:

在瀏覽器中輸入URL

step2:

瀏覽器解析URL中包含的信息。包括協議（「https」），域名（「github.com」）和資源（「/」）。 在這種情況下，「.com」之後沒有指示特定的資源，所以瀏覽器知道檢索主（索引）頁面

step3:

瀏覽器與ISP進行通信，對主機的Web伺服器的IP位址進行DNS查找www.github.com.
DNS服務首先聯繫根伺服器, 查詢 https://www.github.com 頂級域伺服器的IP位址。
該地址被發送回您的DNS服務。
DNS服務與「.com」名稱伺服器進行另外的溝通，並請求 https://www.github.com. 的地址。

step4:

一旦ISP收到目標伺服器的IP位址，它會將其發送到使用者的Web瀏覽器

step5:

使用者的瀏覽器從URL中獲取IP位址和給定的埠號（HTTP協議默認為埠80，HTTPS默認為埠443），並打開TCP套接字連接。
此時，使用者的Web瀏覽器和Web伺服器終於連接了。

step6:

使用者的網絡瀏覽器向網頁伺服器發送HTTP請求，請求 www.github.com. 的主頁面

step7:

Web伺服器接收請求並查找該HTML頁面。
如果頁面存在，則Web伺服器準備響應並將其發送回您的瀏覽器。 如果伺服器找不到請求的頁面，它將發送一個HTTP 404錯誤消息，代表「找不到頁面」

step8: server 端回應

使用者的Web瀏覽器將接收到HTML頁面，然後通過它從上到按下解析尋找列出的其他資源，如圖像，CSS文件，JavaScript文件等。

step9: index.html 頁面

對於列出的每個資源，瀏覽器重複上述整個過程，向伺服器發送HTTP請求。

step10:

瀏覽器完成加載HTML頁面中列出的所有其他資源後，頁面將最終加載到瀏覽器窗口中，並且連接將被關閉

Apache

Apache (2.0) 網頁伺服器

目前市佔率前三的 Web Server 軟體: 『Nginx』、 『 Apache 』、『 IIS 』，

原本 Apache 只是一個開放原始碼的網頁伺服器計畫，但隨著開放原始碼的蓬勃發展，它從只支援網頁伺服器軟體的組織，變成了支援了許多與全球資訊網應用相關的大計畫，並且改名為 Apache Fundation

Workbench

MySQL Workbench 是一款資料庫設計和建模工具，專門為 MySQL 設計。

可以在上面規劃、管理資料庫，最後再利用同步的功能將我們在上面設計的東西推到資料庫上，過程中一行程式碼都不用打，因為都幫我們處理好了

Workbench 會自動生成對應的 MySQL 語法，因此 Workbench 內 Preferences 的 MySQL 版本以及目標的 MySQL Server 版本就很重要。

PhpMyadmin

hpMyAdmin則是用來管理MySQL的一個網頁版工具。

架設軟體種類

常見的有 xmapp,mamp,appserv…

XMAPP

XAMPP是個自帶管理介面的整合包，把各項架站所需都打包好了。XAMPP裡的X指跨平台、A是Apache、M是MySQL（或者MariaDB，一個MySQL的分支），兩個P分別代表PHP和Perl。

MAMP

MAMP是以MacOS、Apache、MySQL及PHP or Python的字首結合的開發工具，但也有支援Windows系統，適用大多使用者。

AppServ

Appserv架站包簡單的融合了Apache、PHP、MySQL等網站環境元件
