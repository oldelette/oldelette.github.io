Decorator

Ref1 Ref2 Ref3

由於 Python 的基本語法過於簡潔，所以大部分的設計與技巧都會為了強化其架構的依賴性與開發性，而讓 Python 的語法變得比原先的繁複。
這次要介紹一個少數「化繁為簡」的語法 — — Decorator ！

Decorator 被大量廣泛的使用在各方 library/package 中，具有幾個最主要的優點：

  • 靈活度高
  • 易讀性高
  • 協助封裝效果好
  • 程式碼重複率低/簡潔度高

Decorate 範例

  • Example

這個例子中的裝飾器 p_decorate 將 get_text此函式作為參數取用，並在修飾後輸出了一個新的函式。另外，也可以注意一下在裝飾器中定義的函式func_wrapper會取用get_text中的name












def get_text(name):
   return "lorem ipsum, {0} dolor sit amet".format(name)

def p_decorate(func):
   def func_wrapper(name):
       return "<p>{0}</p>".format(func(name))
   return func_wrapper

my_get_text = p_decorate(get_text)

print(my_get_text("John"))
  • 改用 Decorate 改寫如下










def p_decorate(func):
   def func_wrapper(name):
       return "<p>{0}</p>".format(func(name))
   return func_wrapper

@p_decorate
def get_text(name):
   return "lorem ipsum, {0} dolor sit amet".format(name)

print(get_text("John"))

數個 decorate

透過一般的方式，裝飾器可能會以下方的形式使用：
get_text = div_decorate(p_decorate(strong_decorate(get_text)))















def p_decorate(func):
   def func_wrapper(name):
       return "<p>{0}</p>".format(func(name))
   return func_wrapper

def strong_decorate(func):
    def func_wrapper(name):
        return "<strong>{0}</strong>".format(func(name))
    return func_wrapper

def div_decorate(func):
    def func_wrapper(name):
        return "<div>{0}</div>".format(func(name))
    return func_wrapper
  • 藉由python的語法糖，decorator可以以更簡潔的方式呈現：







@div_decorate
@p_decorate
@strong_decorate
def get_text(name):
   return "lorem ipsum, {0} dolor sit amet".format(name)

print(get_text("John"))

Class Decorator

Decorator 除了有 function decorator ，也有 class decorator




















class Dog:
    def __init__(self, func):
        self.age = 10
        self.talent = func

    def bark(self):
        print("Bark !!!")

@Dog
def dog_can_pee():
    print("I can pee very hard......")


if __name__ == "__main__":
    dog = dog_can_pee

    print(dog.age)
    dog.bark()
    dog.talent()

由上述範例可以得知，當我們的 decorator 是一個 class decorator 時，要傳入的 function 主體 dog_can_pee 就會從 class 裡頭的 __init__ initializer 被吃進去，然後執行你想操作的動作：
在這個例子裡，我將傳入的 function dog_can_pee 以 assignself.talent 的方式宣告為 class 的 instance method。

Python decorators vs inheritance

繼承（Inheritance）

  1. 繼承是靜態的，我們不能在執行中動態地改變物件的行為，只能更換成另一種物件。
  2. 在多數的物件導向語言中的繼承，一個類別只能有一種父類別。

Closure

Ref1-Lambda Function 與 Closure 之謎！
Ref2-淺談Python閉包和late binding機制

a closure (also lexical closure or function closure) is a technique for implementing lexically scoped name binding in a language with first-class functions

Closure 有很廣泛的應用，我們可以用閉包來減少全局變量的定義進而提供數據隱藏，還可以提供面向對象的編程風格，使代碼實現更優美。

先了解 嵌套函數 (nested function) 和非局部變量 (non-local variable) 的概念

  • nested function: 定義在函數內部的函數
  • non-local variable: 在 nested fuction 訪問外層函式的 local variable 稱為 non-local variable.

printer 是一個 nested function， msg 就是 printer 的 non-local variable。










def print_function(msg):
	# printer is a nested function
    def printer():
        print(msg)
    return printer
    
p = print_function("Hello world!")
p()
print("p: ",p.__closure__)

return printer 語句返回的不僅僅是一個函數，還有一個閉包，這個閉包裏保存了函數要用到的所有non-local variable (非局部變量)

舉例說明 closure

在一般認知當中，function 中的 variable 其 life cycle（生命週期）會隨著 function 執行完而消滅。
下面例子中，variable height 明明是在 function dog() 的 scope 裡，理論上在 12 行 call 完 dog() 後就要消失，怎麼會在 13行 call dog_profile() (也就是 call profile() 啦)時，還能夠找到 variable height 呢？！
原因是出在 return functionprofile 時，function profile 偷偷 capture 住 variable 了，也就是把原本屬於別人上一層的變數偷渡到自己的 function scope ，而被捕捉的 variable 就是 captured variable！而帶有 captured variable 的 function 我們就稱為 closure (function)！

以上就是所謂的 closure – 「把別人（上一層）的 variable 偷渡關進自己的 function scope」














def dog():
    height = 40
    
    def profile():
        print("I'm a dog and my height is {}.".format(height))
        
    return profile



if __name__ == "__main__":
    dog_profile = dog()
    dog_profile()

Binding

late binding 機制

Python’s closures are late binding. This means that the values of variables used in closures are looked up at the time the inner function is called.

LEGB

根據 Python.org 的 tutorial 中的說明，當一個變數被使用時，會遵循 LEGB 的規則，也就是 Local、Enclosing、Global 與 Builtins







glob = 3
def func(x):
    y = x + glob
    def inner():
        return y + 1
    return inner, abs
  • Local，在 local variables 裡查找。
  • Global，是 global variable。
  • Enclosing，也是 enclosing scope。
  • Builtins，也就是從 builtins 模組裡去找，上面的例子裡就是最後用到的 abs 。

當以上都找不到這個你要的變數時，就會引發 NameError 。

Function Scope

Python 中當你定義一個 function 時，你就創造了一個 scope。
這個 scope 會影響到你這個 function 裡所有 local 與 non-local 變數會如何被參照

* 和 ** 為可變形參數

用於 function params 時

  • 預設參數



def plus(a, b, c=None):
    res = a + b + (c if c else 0)
    return res
  • *－收集至 Tuple
    透過 * 收集的引數會被放到一個 tuple 中，所以我們可以使用 for 來對它進行迭代。





def plus(*nums):
    res = 0
    for i in nums:
        res += i
    return res

用於 function arguments 時

假設有個 function intro:





def intro(name, age, city, language):

    ))

給一組 list 的 data lst 和 dict 的 data dic:



lst = ['dokelung', 27, 'Taipei', 'Python']
dic = {'name': 'dokelung', 'age': 27, 'city': 'Taipei', 'language': 'Python'}

使用 * 和 **:



test(*lst)  # I am dokelung. I am 27 years old. I live in Taipei. I love Python
test(**dic) # I am dokelung. I am 27 years old. I live in Taipei. I love Python
