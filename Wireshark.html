Following Protocol Streams

It can be very helpful to see a protocol in the way that the application layer sees it.
Perhaps you are looking for passwords in a Telnet stream, or you are trying to make sense of a data stream.
Maybe you just need a display filter to show only the packets in a TLS or SSL stream.
If so, Wireshark’s ability to follow protocol streams will be useful to you.

Fllow tcp stream

跟蹤TCP傳輸控制協議的通信資料段，將分散傳輸的資料組裝還原

Wireshark針對通訊協定提供了完整的Follow TCP/UDP Stream功能，能夠依序追蹤網路封包所傳送的內容，此種方式可以將有關聯性的網路封包一起呈現，如此在進行網路行為或通訊協定的行為分析時，將更為清楚易讀。

Traffic from the client to the server is colored red
Traffic from the server to the client is colored blue

TLS 加密

TLS: tcp , port 443

TLS加密的流程

  1. Client Hello
    (client 告訴server要連接)
    提供我的電腦可以接受的加密通訊協定版本、一個用戶端生成的亂數、支援的加密演算法及支援的壓縮演算法

  2. ServerHello Finished
    (server回覆 client)
    代表收到了client電腦傳過去的招呼, server回傳確認使用的加密通訊協定版本（若是伺服器能提供的最高版本比客戶端來得低的話，就無法建立連線）、一個伺服器生成的亂數、確認使用的加密方法及伺服器憑證

    伺服器告知用戶端，接下來將會依照雙方協議的加密方法等連線參數來進行加密，並傳送 Finished訊息。

  3. ClientKeyExange Finished
    用戶端告知伺服器，接下來將會依照雙方協議的加密方法等連線參數來進行加密，並傳送Finished訊息

Libpcap 檔案格式

Libcap 作爲捕獲網絡通信數據的事實上的標準工具
pcap是數據包捕獲的縮寫，一般是tcpdump或wirehark抓包後存取的文件。

pcap

  1. 24Byte 的Global Header (文件頭)
  2. 16Byte 的數據標頭 + 標頭對應數據包 (Packet Header+Packet Data)
  3. 下一個 16Byte 的數據標頭 + 標頭對應數據包
  4. 下一個 16Byte 的數據標頭 + 標頭對應數據包
  5. 一直重複到文件尾巴…

前面24byte說明 (Global Header)

  • Magic：4Byte：0x1A 2B 3C 4D:用来標示文件的開始
  • Major：2Byte，0x02 00:當前文件的版本號碼 (默認值為0x2)
  • Minor：2Byte，0x04 00:當前文件的次要版本號 (默認值為0x4)
  • ThisZone：4Byte 當地的標準時間；全零
  • SigFigs：4Byte 時間戳精度；全零
  • SnapLen：4Byte 最大的存儲長度
  • LinkType：4Byte link 類別

VPN (Virtual Private Network)

https://nordvpn.com/zh-tw/what-is-a-vpn/

當嘗試進入網際網路上的網站時，可以透過連線到網際網路服務供應商 (ISP) 來開始。他們會將您重新導向到任何您想要瀏覽的網站 (或其他線上資源)。您所有的網際網路流量都會經過您的 ISP 伺服器，表示他們可以看到您在線上所做的並記錄一切。他們甚至會將您的上網資料交給廣告商、政府機構和其他第三方。

這就是 VPN 發揮作用的地方。您的網際網路流量將會特別設定的遠端伺服器重新導向。這樣 VPN 可以隱藏您的 IP 位址，以及加密您傳送或接收的所有資料。對於攔截這些加密資料的人而言，加密資料基本上都是亂碼 — 因此無法讀取。

  • vpn 是加密流量

原理

https://sites.google.com/site/vpnjianjieyuqiyingyong/xian-kuang-miao-shu/vpn-yuan-li

  1. Tunneling
    穿隧技術其實很簡單，其原理就是將你原本的封包(Packet)視為資料(Data)再重新封裝(Encapsulation)，變成新的封包傳輸在公用網路中。

  2. Encryption & Decryption
    VPN各個節點間是利用3DES或DES這種對稱式金鑰加密，以達到高傳輸效率。而VPN對於身分認證跟金鑰傳輸是利用非對稱加密的方式。

  3. Key management

現行常用密鑰管理的技術又可分為SKIP(Simple Key management for IP)與ISAKMP/Oakley (又稱為IKE)兩種。

SKIP: 利用Diffee-Hellman演算法則， 應用在網路上傳輸密鑰的技術
IKE: Oakley定義如何分辨及確認密鑰，ISA KMP定義分配密鑰的方法。
  1. Authentication

這個部分是為了確認跟你連接或你所連接VPN對方的身分是不是本人所建立起的機制，主要分為兩種：
a. 使用者身分鑑別部分：使用者名稱跟密碼、IC卡鑑別‧‧‧等。
b. 設備身分鑑別部分：CA的X.509憑證、分享金鑰。

Proxy

Proxy是「代理」的意思。最早應是出現在網路的Firewall (防火牆)功能中。
防火牆簡單的說是用來保護網路的安全，在你的內部網路與外部網路之間建立一道像牆般
的保護，使內外有所區隔分別，而所有的進出、連線動作都必需要經過這道牆。因此對外
面網路世界中的機器來說，它們只看的到這道牆，而看不到這牆內的機器，而牆內的機器
也必需透過這道牆才能出去，因此這道做為牆的機器我們就可以稱它為Proxy Server

Wireshark

Wireshark 是一種協定分析軟體，即“資料封包監聽”應用程式，適用於網路故障排除、分析、軟體和協定開發
以及教學。當資料流透過網路來回傳輸時，監聽軟體可以“擷取”每個協定資料單元 (PDU)，並根據適當的 RFC 或其它規範對其內容進行解碼和分析。

封包內容的byte數

每一層都有一個欄位指向上一層，表明上一層是什麼協議。這大概是因為發包的時候會在數據上依次加上應用層、傳輸層、網絡層、鏈路層的頭部，但是對方收到數據包後是從最底層（鏈路層）開始層層剝去頭部解包的，所以在每層上有一個欄位指向上層，表明上層的協議，對方就知道下一步該怎麼解包
