Switch & Router (NAT)

必看影片: 集線器，交換機和路由器解釋 - 有什麼區別？

  • Hub and Switches are used to exchange data within a local area network.
  • Not used to exchange data outside their own network.
  • To exchange data outside their own network, a device needs to be able to read IP addresses.
  • Hub & Switches are used to create networks
  • Routers are used to connect network

Hub 跟 Switch 不能用來跟外往交換數據，因為要跟外界互聯網交換數據，需要能夠讀取 IP address 的設備， 而剛好 Hub 跟 Switch 不能夠讀取 IP Address.
Router 解決跟外網連接的問題。
Hub 跟 Switch 用來創造網路，而 Router 用來連接這些網路。

HUB

集線器（Ethernet hub）是指將多條乙太網路雙絞線或光纖集合連接在同一段物理媒介下的裝置。
線器是運作在OSI模型中的實體層[1]，可以讓其連結的裝置工作在同一網段。

HUB有兩大特性，一個就是廣播，一個就是半雙工。

Disadvantage

由於集線器會把收到的任何數位訊號，經過再生或放大，再從集線器的所有埠送出，這會造成訊號之間碰撞的機會很大，而且訊號也可能被竊聽，並且這代表所有連到集線器的裝置，都是屬於同一個碰撞網域以及廣播網域，因此大部份集線器已被交換機取代。

  • 實體 HUB機器照片

Switch

交換器
是一種網路硬體，通過報文交換接收和轉發資料到目標裝置，它能夠在電腦網路上連接不同的裝置。
和 HUB看起來一樣，但實際上差別很大。
首先 switch並不一直廣播，而且是全雙工的。

主要是 SWITCH 會記錄封包中的MAC位址
所以當 電腦A 傳送資料給 電腦B 時，其他電腦並不會也收到資料，而且這個時候別的電腦也可以同時互相傳送資料。
相比於 hub 安全許多，也節省 hub 不必要的網路流量。

  • 實體 Switch機器照片

Router

又稱為路由器或IP分享器或寬頻分享器

  • Router data from one network to another based on their IP address.
  • The Router is the gateway of a network.

下圖顯示 router 當作 gateway 使用， 只有紅色的資料能夠送進這個 red network.

路由器最主要的工作就是決定要將網路封包送往何處，而為了要達到這樣的目的，路由器至少必須做到以下這些事情：

  1. 知道目的端的位址在哪裡。
  2. 標示出來源端的位址，並且把它學習起來。
  3. 尋找這個封包可能要送往的路徑有哪些。
  4. 從可能的路徑中選出最佳路徑。
  5. 維護並更新這些路由所需的資料。
  • 實體機器照片

NAT 功能

Network Address Translation

一般IP分享器LAN port裡的電腦是使用虛擬IP，也就是俗稱的假IP。
這個網段通常是192.168.X.X，最常見的是192.168.0.X與192.168.1.X。這個IP網段是保留的網段，在實際網際網路並不能使用。
NAT這個功能負責記錄網卡MAC位址與假IP的關係並做轉換。

Example

舉例來說，假設使用者有兩台PC透過IP分享器上網

A電腦IP為192.168.0.2，網路卡MAC位址是00046F12301A

B電腦IP為192.168.0.50，網路卡MAC位址是000879215B09

當A電腦想要連網站yahoo.com.tw時，A電腦會送出資料給yahoo，而IP分享器會記錄192.168.0.2的MAC是00046F12301A，並把資料內來源IP 192.168.0.2改成WAN port的真實IP，然後送往YAHOO。Yahoo收到以後會依照真實IP位址回傳資料給IP分享器，IP分享器再依照資料內的MAC位址判別出這封包是要給A電腦的，於是把目的IP改成192.168.0.2送往A電腦。

簡單的說，IP分享器是透過傳送封包內的MAC位址來分辨這個封包資料是要給哪一台電腦的，進而達到分享IP的功能。

Modem

影片 Modem vs Router - What’s the difference?

Modem 全名為 modulator-demodulator 中文「數據機」
由網絡供應商提供的調解器，其功能就是把互聯網的類比訊號調解為電腦可讀取的數碼訊號，而 Modem 同樣地亦可把電腦的數碼訊號轉化為互聯網的類比訊號。

  • A computer only reads digital signals
  • The internet only reads analong signals.
  • Modem demodulates incoming analong signals into a digital signal. (internet -> computer)
  • Modem modulates outgoing digital signals into an analong signal. (computer -> internet)

  • Most routers will have a built-in switch inside of them.

網卡

網卡的主要功能是將網絡數據封包發送並將接受的網絡數據包解封並打包成完整的數據，在傳輸過程中是不能傳輸太大的數據，必須將數據分解成若干小數據包，再將這些數據包連續的傳輸出去，在打包數據的時候，為了保持數據的連貫性，會在打包的數據的首尾加上標識，這樣在解包的時候，就可以對應首尾的標識，來將相應的數據接起來。

網卡的種類

  1. 獨立網卡
  2. 集成網卡
  3. USB網卡
  4. USB無線網卡

MAC

Media Access Control
網路介面卡都有一個獨一無二的識別碼，這個識別碼是由六組16進位數字組成的物理位置，也稱為MAC（Media Access Control）位址。
這個位址分為兩個部分，前三組數字是廠商ID；後三組數字則是網路卡的卡號，理論上全世界沒有兩張網路卡的 MAC位址是相同的。

Cisco Switch

Cisco Switch有多種命令模式，包含User Mode、Privileged Mode、Global Configuration Mode 及 Interface Configuration Mode等等，不同的模式代表不同的用途和不同的權限等級，而各種不同的模式切換有其關聯性，並不能隨意從任意兩者之間來切換。

一開始所使用的是User Mode，User Mode只具備一般簡單的指令，大部分都限制在只能使用用來檢測的指令，而不能更改設備的設定值，當然就是代表權限最低的指令模式。

Privileged Mode則可以執行用來設定 Cisco設備的指令。
從User Mode進入Privileged Mode的指令是enable

進入Privileged Mode之後，可以再進入Global Configuration Mode，大致而言，Global Configuration Mode是針對Switch整體的設定

configure terminal

進入Global Configuration Mode之後，提示符號會變成(config)#
這時候還可以從Global Configuration Mode再進入Interface Configuration Mode，此時要使用何種指令，決定在於要進入到哪個介面

Protocol

SNMP

Simple Network Management Protocol 簡易網路管理通訊協定

是目前網路上用來進行網路管理的通訊協定，最主要的目的在於管理網路上各式各樣的設備。

SNMP 主要分為三個元件

  1. 管理端（Manager）
  2. 代理者（Agent）
  3. 管理資訊庫（Management Information Base，MIB）
一個 Manager 可以管理網路中的多個設備。
而每台設備上運行著 SNMP Agent 用於和 Manger資訊交互，交流的內容需要符合 MIB 的規範。

Manager

Manager 通常被稱為網路管理工作站（Network Management Station，NMS）
利用SNMP通訊協定向代理者（Agent）查詢所需的相關資訊，如網路設備運作狀態、系統硬體的配置（如CPU使用率、硬碟利用率）等。

  • SNMP 所定義的五種Manager與Agent溝通指令

Agent

代理者（Agent）通常是一個執行程式（通常運作在被監控的設備上），負責讀取被監控設備上的相關資訊，而後接收到管理端所發出的SNMP Get-request、Get-next-request等查詢指令時，再將相關資訊回傳至管理端。

MIB

Management Information Base 資訊管理庫

但在現實環境中，每種網路或設備對其資料的表達方式也存在大小不一的差異，因此必須採用一套抽象的語法來描述所有類型的資訊。

所以，SNMP定義了網管資訊庫（簡稱為MIB），利用階層性的描述，說明所有受管理設備資訊的屬性，並將這些設備稱為SNMP物件（Object）。

SNMP 缺點

  • 性能不足，在下發和讀取配置時，採用依次讀取，效率低。
  • 基於UDP協議傳輸，可靠性較差，只能依靠本身機制保證可靠性，影響性能；
  • 私有MIB導致多廠家設備統一管理難度大。

正是由於SNMP的各種不足和缺陷，才導致了NETCONF的產生。

  • NETCONF與SNMP對比

NETCONF

The Network Configuration Protocol

NETCONF 協議提供了一種更簡單的方式來管理（”查詢，配置，修改，刪除”）設備，就像資料庫操作中的 DML. 同時開放了 API 介面，當想要對設備進行操作時，直接通過調用 API 進行

NETCONF 採用 C/S 的架構，通過 RPC 在 client 和 server 間交流。

  • Client 可以是 Python 腳本或應用
  • Server 一般指的是網路設備

在具體實現上有三個組件：

  1. NETCONF agent:
    運行在網路設備上，用於接收和處理 RPC 請求。還可主動將一些告警事件通知客戶端。
  2. NETCONF client：
    利用 NETCONF 協議對網路設備進行管理以及接收 agent 發出的告警通知。
  3. datastore：
    在 NETCONF 中，區分了多個不同類型的 datastore, 這些 datastore 保存著不同狀態下的設備資訊

NETCONF 分為 4 層，各層之間項目獨立。

1. 內容層：

這一層包含了以 XML 或 JSON 格式的配置數據，也就是想對設備進行管理的具體內容。（由 XSD 或者 YANG 約束生成）

2. 操作層：

定義了 Client 和 Server 交互時的一系列操作方法，用於獲取或修改配置數據。

3. 消息層：

為編碼數據時，提供了一種 RPC 和通知的機制

* RPC invocations(<rpc> messages)  
* RPC results(<rpc-reply> messages)
* event notifications(<notification> messages)

4. 傳輸層：

NETCONF 使用 SSH 或 TLS 協議，保證數據在 Client 和 Server 傳輸的安全性。

  • NETCONF 交互

RESTCONF

RESTCONF 是通過 REST 來實現對網路設備管理的協議。
其本質和 NETCONF 很像，使用 YANG 進行數據的定義和約束，使用 HTTP 進行交互。使用 NETCONF 中 datastore 的概念，進行資訊的儲存。

REST – Representational State Transfer (表現層狀態轉化)
是建立在 HTTP 基礎上，對其進行規範的一種架構風格要求。

下圖中很好的表示了 RESTCONF 協議組成，很形象的指出 RESTCONF = NETCONF / YANG + HTTP(s).

VLAN 技術

Virtual LAN 虛擬區域網路

利用某一特定技術，將某些工作站(或電腦)組織設定一個『邏輯網路』(Logical Network)。
經過 VLAN 技術的處理，一個網路區段 (或稱 IP 子網路) 裡可再區分成多個虛擬區域網路(VLAN)

上圖利用 VLAN 技術將此 12 部電腦組合成三個虛擬區域網路，被組合 VLAN 電腦可以不是同一 Switch 底下的電腦，可以跨接其他 Switch下電腦的組合。

譬如 VLAN0 中的 PC0、PC1 在 Sw0 上，又 PC4 與 PC5 在 SW1 交換器上

Q: VLAN 為何會這樣切?
A: 大多為了劃分工作群組，也就是同一 VLAN 內的電腦通訊非常頻繁，期望他們之間通訊不要影響到其他電腦。假設 PC0 是一個資料庫伺服器，專供應 PC1、PC4、PC5 電腦使用，其他電腦甚少使用到，則將他們組合一個 VLAN。

VLAN Advantage

規劃虛擬區域網路的優點:

  • 減抵『廣播網域』(Broadcast Domain)
  • 具有『防火牆』(Firewall) 功能
  • 劃分『工作群組』(Workgroup)
  • 散居『子網路區段』位置
